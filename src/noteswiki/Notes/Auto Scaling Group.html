#compute - In real life load on applications varies. - In the cloud we can create and dismantle servers very quickly - The goal of Auto Scaling Group ASG is to: - Scale out (Add EC2 instances) to match an increased load. - Scale in (Remove instances) to match a decreased load - Ensure we have a min/max number of machines to match load. - Automatically register new instances to a load balancer. - Replace unhealthy instances - Cost savings: only run an optimal capacity - ASGs can also work very well with load balancers. -
