### Overview - Route53 is a managed DNS - DNS is a collection of rules and records which helps clients understand how to reach a server through URLs - In AWS the most common records are: - 'A' Record (IPv4) - AAAA IPv6 - CNAME: hostname to hostname. - Alias(For AWS services ELB, CloudFront etc) - ### Route53 Routing policies - *Simple Routing Policy* (No health Checks) - *Weighted Routing policy* (Kind of like load balancing) - *Latency Routing Policy* (Based on latency to closest node) - *Failover routing policy*
